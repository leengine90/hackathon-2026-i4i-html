01
서비스 개요

1.1 서비스 소개

마이 스마트 루틴 - 뿌듯은 초등학생이 매일 스스로 공부를 시작하고 계속 이어갈 수 있도록 돕는 자기주도 학습 루틴 서비스입니다. 보호자(설계자)가 루틴 구조를 만들고, 아이(실행자)가 그 안에서 실행합니다. 하루 3단계 루틴(하루 설계 → 실행 → 하루 정리), AI 캐릭터 생성, 우주 마을 짓기 보상, 가족·친구와 함께하는 소셜 기능을 결합하여 아이가 꾸준히 공부 습관을 만들어 나갈 수 있도록 지원합니다.

1.2 핵심 가치

"매일 스스로 시작하는 아이로 자라나는 것, 그 첫 걸음을 함께합니다."

핵심 인사이트

아이가 공부를 못 시작하는 이유는 "하기 싫어서"가 아니라 "어디서 시작할지 막막해서"입니다.

  • 선택지를 주면 아이도 2분 안에 오늘 계획을 완성한다
  • 짧은 성공 경험이 쌓이면 스스로 하고 싶어진다
  • 혼자보다 같이할 때 더 오래 지속된다

3가지 핵심 가치

  • 습관이 실력을 만든다 — 초등 시기엔 꾸준히 하는 습관이 먼저
  • 재미있어야 매일 한다 — AI 캐릭터·우주 마을 짓기로 내재 동기 형성
  • 함께하면 더 오래 간다 — 가족·친구와의 공동 달성 구조

핵심 혁신 & 문제 정의

2.1 우리가 발견한 것

기존 습관 앱(Duolingo, Forest, 열품타 등)은 모두 같은 전제를 공유합니다. "습관을 만들 사람이 직접 그 구조를 설계한다." 이 전제는 성인에게는 작동하지만, 만 6~9세에게는 성립하지 않습니다. 기존 앱들은 이 발달 단계를 무시하고 성인용 습관 앱을 아이 버전으로 만들었을 뿐입니다.

03
3가지 핵심 혁신

우리가 발견한 것 → 우리의 대답 → 결과. 세 혁신은 하나의 서사로 연결됩니다. 상세 내용은 [별첨 A] 참조

혁신 A
설계자와 실행자의 분리 — Designer-Executor Separation
세상의 모든 습관 앱이 공유하는 전제를 깬 유일한 구조

기존 앱 — 사용자가 직접 설계하고 실행

초등생이 "오늘 뭐 할지" 결정하는 과정 자체가 가장 큰 장벽. 저학년의 33.2%는 시작을 스스로 못 함.

뿌듯 — 보호자가 설계, 아이가 실행

아이는 루틴 순서 조정만 가능. 결정 피로 없이 바로 시작. 발달 단계에 맞는 유일한 구조 분리.

혁신 B
습관 인프라 — Habit Infrastructure
콘텐츠 경쟁이 아닌, 모든 학습 위에 깔리는 구조

기존 앱 — 콘텐츠 완결형

Forest 안에서 공부, Duolingo 안에서 영어 학습. 홈런, 학습지와 경쟁 관계.

뿌듯 — 인프라형

콘텐츠 없이 구조만 제공. 학원·학습지가 뿌듯 위에서 더 잘 작동. 공존 가능.

혁신 C
실패에 친절한 설계 — Failure-Friendly Design
단기 리텐션을 위해 아동의 장기 학습 동기를 훼손하지 않습니다

기존 앱 — 패널티형

Forest 식물 소멸, Duolingo 하트 감소, 열품타 랭킹 하락. 성인에겐 작동하지만 초등 저학년에겐 공부 회피 정서를 형성할 수 있음.

뿌듯 — 친절형

연속 기록 보호 쿠폰, 격려 알림, 순위 없음. 실패에 친절한 설계는 기능이 아니라 교육 철학입니다.

05
서비스 설계

5.1 타겟 사용자 (페르소나)

🧒 민준 (초등 3학년, 9세)

학원은 다니지만 집에서 혼자 공부를 시작하지 못함. 유튜브를 보다가 공부 타이밍을 놓치고, "오늘 뭐 해야 하지?"를 스스로 결정한 적이 없음.

필요한 것
  • 시작을 도와주는 친근한 가이드
  • 달성했을 때의 성취감
  • 친구와 함께하는 재미

👩 민준 엄마 (35세, 직장인)

매일 "공부해"라고 말하기 싫음. 아이가 스스로 하는 습관을 만들어주고 싶지만 방법을 모름. 오늘 뭘 했는지 간단히 확인하고 싶음.

필요한 것
  • 잔소리 없이 아이 스스로 하게 하는 구조
  • 오늘의 공부 현황 간단 확인
  • 아이에게 응원을 전달하는 방법

06
핵심 기능 상세

본 섹션은 뿌듯 v1.0 인터랙티브 프로토타입 기준으로 작성되었습니다. 실제 구현된 화면과 인터랙션을 기반으로 기술합니다.

6.1 온보딩

온보딩은 보호자가 먼저 완료하는 필수 설정으로 구성됩니다. 총 10단계(스텝 1~8, 10)로 이루어지며, 각 단계는 진행 바와 뒤로가기 버튼을 통해 앞뒤로 이동할 수 있습니다.

① 온보딩 플로우 — 전체 구조

스텝화면명내용비고
스플래시시작 화면'뿌듯' 로고 + 캐릭터 이미지 + 시작하기 버튼시작 화면 디자인 시안 보기(데모용) 진입 가능
Step 1앱 주체 확인보호자 / 아이 선택. 아이 선택 시 "부모님이 먼저 설정해줘야 해요" 안내 표시보호자 선택 시에만 다음 단계 진행
Step 2아이 기본 정보아이 닉네임 입력 + 학년 선택 (1~3학년)닉네임 입력 필수
Step 3학습 환경 파악학원 수 선택 (없음 / 1~2개 / 3개 이상)단일 선택
Step 4어려움 유형 선택공부 시작 안 함 / 꾸준히 못 함 / 스스로 안 함 / 어떻게 도와줄지 모름 — 복수 선택 가능최소 1개 선택 필수
Step 5아이 성향 찾기3문항 질문 (선택 사항, 건너뛰기 가능)설정 화면에서 재진행 가능
Step 6루틴 직접 설정루틴 이름·양·단위·빈도를 테이블 형태로 입력. 기본값 3개 제공, 최대 5개까지 추가 가능루틴 이름 입력 필수
Step 7루틴 크기 확인"딱 좋아요" / "좀 줄일게요" 선택. '줄일게요' 선택 시 쪼개기 팁 카드 노출단일 선택
Step 8알림 시간 설정오전/오후 토글 + 시(1~12시) + 분(0·10·20·30·40·50분) 그리드 선택매일 알림 시간 설정
Step 10성향 결과 & 모드 추천성향 결과 멘트 표시 + 소셜 모드 선택 (가족과 함께 / 친구와 함께 / 혼자 집중). '뿌듯 시작하기' 버튼으로 온보딩 완료모드는 언제든지 설정에서 변경 가능
온보딩 완료 후 캐릭터 만들기 화면으로 이동하며, 이후 학습 환경 설정 완료 중간 페이지(아이 이름·학년, 루틴 수, 알림 시간 요약 표시)를 거쳐 홈 화면으로 진입합니다.

6.2 오늘 루틴 (홈 화면)

홈 화면은 하루 3단계 루틴 사이클(하루 설계 → 실행 → 하루 정리 → 완료)로 구성됩니다. 화면 상단에는 현재 보유 씨앗 수와 연속 달성 일수가 항상 표시됩니다.

① 하루 설계 (design 단계)

온보딩에서 설정한 루틴 목록이 2열 그리드로 표시됩니다. 각 카드에는 루틴 이름, 이모지, 목표 양(예: 5문제)이 표시되며, 탭하면 선택/해제됩니다. 최대 3개까지 선택할 수 있으며, 1개 이상 선택 시 "공부 시작!" 버튼이 활성화됩니다.

기본 루틴 3개(수학 문제집 5문제, 영어 단어 10개, 독서 20분)가 디폴트로 제공됩니다.

② 실행 (execute 단계)

선택한 루틴 카드가 목록으로 표시되며, 각 카드에는 세 가지 액션 버튼이 있습니다.

버튼동작
⏱ 타이머해당 루틴의 타이머 시작. 화면 상단에 경과 시간 표시 (MM:SS 형식)
✅ 완료즉시 완료 처리. +10 씨앗 적립, 별 파티클 애니메이션 재생

6.3 AI 캐릭터 & 마을 짓기

씨앗은 앱 내 기본 화폐입니다. 공부 완료 시에만 지급되며, 앱 접속이나 광고 시청으로는 지급하지 않습니다. 모은 씨앗으로 우주 마을을 짓고 건물을 구매합니다.

🪐 우주 마을 세계관

내 우주인 캐릭터가 우주 속 행성 땅에서 시작합니다. 씨앗을 심으면 마을이 생겨나고, 집이 지어지고, 우주 마을이 점점 커져나갑니다.

시작: 씨앗 행성 (빈 우주 행성) → 성장: 새싹 마을 → 꽃 마을 → 나무 도시 → 목표: 우주 문명

나의 우주 화면 구성

나의 우주 화면은 3개 탭으로 구성됩니다.

  • 우주 탭: 현재 마을 레벨·이름, 다음 레벨까지 진행 바, 연속 달성 일수, 총 달성일, 보유 씨앗 표시. 가로 스크롤 마을 뷰에서 구매한 건물이 표시됩니다.
  • 상점 탭: 씨앗 적립 방법 테이블 + 씨앗 사용 테이블 + 카테고리별(건물·장식·특별) 구매 가능 아이템 목록. 씨앗이 부족한 아이템은 흐리게 표시됩니다.
  • 뱃지 탭: 달성한 뱃지(3일 연속, 첫 완료, 독서왕 등)와 미달성 뱃지를 구분하여 표시합니다.

8.1 v1.0의 현실적 한계

① 자기보고 기반의 완료 처리

현재 루틴 완료는 아이가 직접 ✅ 버튼을 누르는 방식입니다. 실제로 공부했는지 검증할 수단이 없어 신뢰도에 한계가 있으며, 아이가 습관적으로 완료만 눌러버리는 패턴이 생길 수 있습니다.

② 루틴의 단방향성

보호자가 루틴을 설계하고 아이가 실행하는 구조는 강점이지만, 실제 학습량·난이도 데이터가 앱 내부에서만 순환됩니다. 외부 학습 시스템과 연결되지 않으면 앱이 별도의 섬처럼 작동하게 됩니다.

③ AI 기능의 프로토타입 수준

v1.0에서 AI 알림 최적화, 하루 정리 질문, 소셜 모드 추천은 규칙 기반 또는 고정 로직으로 구현되어 있습니다. 실질적인 개인화는 누적 데이터가 쌓인 이후에야 가능합니다.

[별첨 A] 핵심 혁신 상세 & 기존 서비스 분석

A.1 발달심리학적 근거

Vygotsky의 ZPD 이론

만 6~9세는 메타인지(자신의 행동을 계획하고 모니터링하는 능력)가 아직 형성 전 단계입니다. 이 시기 아이에게 스스로 계획하라는 요청은 발달 단계에 맞지 않습니다. 뿌듯의 설계자-실행자 분리는 이 이론에서 출발합니다.

Erikson 발달 이론

만 6~9세는 근면성 대 열등감(Industry vs. Inferiority)의 시기입니다. 반복적 실패 경험과 죄책감은 앱 이탈을 넘어 학습 자체에 대한 회피 정서를 형성할 수 있습니다. 패널티 없는 설계는 이 이론에 기반합니다.

열품타
공부 시간 측정 + 전국 랭킹
타이머로 공부 시간을 측정하고 전국 랭킹으로 경쟁하는 앱. 수험생·성인 중심 설계.
핵심 메커니즘
타이머 → 시간 집계 → 전국 랭킹 공개
주요 사용자
고등학생·수험생. 10대 후반~20대
차이점
  • 랭킹 비교 → 초등생 열등감·압박감 유발
  • 습관 설계·계획 기능 없음
  • 보호자 연동 전무

Focustown
집중 시간으로 마을을 키우는 타이머
포모도로 방식 기반. 집중 시간이 쌓이면 마을이 성장하는 픽셀 아트 시각 피드백.
핵심 메커니즘
타이머 → 집중 시간 누적 → 마을 성장
주요 사용자
중·고·대학생. 10~20대 전반
차이점
  • 시간 길이에만 집중 → 초등 단위 고려 없음
  • 마을 완성 후 장기 동기 급락
  • 과목·계획 설계 없음

중·고·대학생. 스마트폰 억제 목적
